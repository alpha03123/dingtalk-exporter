# 钉钉聊天记录导出工具 — 使用手册

## 一、功能概述

本工具从钉钉桌面客户端的本地加密数据库中导出聊天记录，提供 Web 界面浏览、搜索和导出。

**核心能力：**
- 解密钉钉本地数据库（V2 / V3）
- 自动处理 `dingtalk.db-wal` 增量日志，补齐尚未写回主库的最新消息
- Web界面浏览全部会话和消息
- 按关键词搜索消息内容
- 支持文本、图片、文件、语音、富文本、审批、引用等20+种消息类型
- 图片在线预览（含引用消息和富文本中内嵌图片自动显示）
- 每4小时自动增量同步新消息
- 勾选会话导出，支持按时间范围筛选（3个月/6个月/1年/2年/全部）
- 导出包含完整附件（图片、文档、PDF等），自包含目录结构
- 导出内容含 agent 友好的 `content` 字段，供 AI 工具参考使用
- 导出目录支持一键 ZIP 打包下载
- 全部本地运行，无需任何云端API调用

**不依赖任何在线API**：所有数据来自本地数据库文件，解密、解析、导出均在本地完成，不需要钉钉开放平台API、不需要网络请求、不需要Token。

## 二、当前环境运行

### 2.1 启动服务

```bash
cd /path/to/dingtalk-exporter
python main.py
```

启动后浏览器访问 **http://localhost:8090**

### 2.2 使用方式

| 功能 | 操作 |
|------|------|
| 浏览会话 | 左侧会话列表向下滚动自动加载更多 |
| 查看消息 | 点击会话名，右侧显示消息流，最新消息在底部 |
| 翻页 | 底部分页栏，点击"更早的消息"查看历史 |
| 搜索会话 | 左侧搜索栏输入关键词实时筛选 |
| 按类型筛选 | 左侧"全部/群聊/单聊"Tab切换 |
| 搜索消息 | 顶部搜索栏输入关键词回车 |
| 查看图片 | 消息中的图片可直接预览，点击放大 |
| 未读标记 | 未读会话显示红色数字角标 |
| 手动同步 | 点击右上角"手动同步"按钮 |
| 勾选导出 | 点击"导出"→ 勾选会话 → 选择时间范围 → "导出选中会话" |
| 全量导出 | 点击"导出"→ 切换到"已导出文件"Tab → "全量导出所有会话" |
| 下载导出文件 | "已导出文件"Tab中点击对应文件的"下载 ZIP"链接 |
| 设为开机自启 | 创建快捷方式放入 `shell:startup` |

### 2.3 消息显示说明

- **消息排列**：按时间顺序，最新消息在最下方，打开对话自动滚动到底部
- **日期分组**：不同日期的消息之间有日期分割线（含星期）
- **富文本/引用消息**：自动查找并显示引用消息中的图片，文字和图片一体显示
- **未缓存图片**：显示为"[图片未缓存到本地]"占位符（该图片未被钉钉客户端下载到本地）
- **文件附件**：已下载到本地的文件（docx/pdf/xlsx等）可直接点击打开

### 2.4 自动同步

服务启动后，定时任务每4小时自动执行：
1. 复制钉钉加密数据库（避免锁冲突）
2. 调用 dingwave 解密
3. 对比上次同步时间，导出增量消息为JSON
4. 更新同步状态

如果钉钉桌面端仍在运行，最新消息可能先停留在 `dingtalk.db-wal` 中。当前版本的“手动同步”会连同 WAL 一起复制、解密并读取；如果同步后仍感觉缺消息，先在钉钉里打开对应会话，再手动同步一次。

## 三、移植到其他电脑或用户

### 3.1 需要携带的文件

```
dingtalk-exporter/
├── config.py              # 需修改（见3.2）
├── main.py
├── decrypt.py
├── parser.py
├── exporter.py
├── attachment.py
├── scheduler.py
├── requirements.txt
├── setup.bat              # Windows 一键安装脚本
├── setup.sh               # Linux/Mac 一键安装脚本
├── tools/
│   └── dingwave.exe       # 解密工具（从 https://github.com/p1g3/dingwave 获取）
└── web/
    ├── api.py
    └── static/
        ├── index.html
        ├── style.css
        └── app.js
```

**不需要携带**：`data/`、`logs/`、`__pycache__/`（运行时自动生成）

### 3.2 配置（大多数情况无需修改）

工具会**自动检测**钉钉用户数据目录：扫描 `%APPDATA%\DingTalk\`、`%LOCALAPPDATA%\DingTalk\` 等多个目录下的 `*_v2`/`*_v3` 文件夹，选择包含数据库文件且最近修改时间最新的用户。绝大多数用户无需任何配置。

直接运行 `python main.py` 即可。如果启动后能正常看到会话列表，说明自动检测成功。

#### 自动检测失败时的手动配置

如果启动报错或看不到会话，可以通过**环境变量**指定（推荐）：

```bash
set DINGTALK_UID=123456789
set DINGTALK_DATA_DIR=C:\Users\username\AppData\Roaming\DingTalk\123456789_v2
python main.py
```

或者直接编辑 `config.py` 中的默认值：

```python
DINGTALK_DATA_DIR = r"C:\Users\<YOUR_USERNAME>\AppData\Roaming\DingTalk\<YOUR_UID>_v2"
USER_UID = "<YOUR_UID>"
```

**如何找到这些值：**

1. 打开钉钉数据目录：
   ```
   %APPDATA%\DingTalk\
   ```
2. 会看到类似 `123456789_v2` 或 `abcdef_v3` 这样的文件夹
3. 如果有多个用户目录，选当前登录用户的那个（修改时间最新的）

### 3.3 目标电脑环境准备

**方式一：一键安装（推荐）**

Windows 用户双击 `setup.bat`，Linux/Mac 用户运行 `./setup.sh`。

**方式二：手动安装**

**步骤1：安装 Python 3.10+**

从 https://www.python.org 下载安装，安装时勾选"Add Python to PATH"

**步骤2：安装依赖**

```bash
cd dingtalk-exporter
pip install -r requirements.txt
```

依赖只有3个包：
- `fastapi` — Web框架
- `uvicorn` — Web服务器
- `apscheduler` — 定时任务

**步骤3：可选获取 dingwave 语音解码工具**

从 https://github.com/p1g3/dingwave/releases 下载对应平台的二进制文件，放入 `tools/` 目录。未安装也可以正常浏览、同步和导出文本/图片/文件消息，只会影响语音消息解码。

**步骤4：确认钉钉数据目录存在**

```bash
# 检查加密数据库文件是否存在
dir "%APPDATA%\DingTalk\<UID>_v2\DBFiles\dingtalk.db"
```

**步骤5：启动**

```bash
python main.py
```

浏览器访问 http://localhost:8090

### 3.4 可选配置修改

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `SYNC_INTERVAL_HOURS` | 4 | 自动同步间隔（小时） |
| `WEB_PORT` | 8090 | Web服务端口 |
| `WEB_HOST` | 0.0.0.0 | 监听地址（改为127.0.0.1仅本机访问） |

## 四、关于API依赖的说明

### 4.1 本工具完全不依赖在线API

| 环节 | 数据来源 | 是否需要网络 |
|------|---------|-------------|
| 数据库解密 | 本地 dingwave.exe 工具 | 否 |
| 消息解析 | 本地 SQLite 数据库 | 否 |
| JSON导出 | 本地文件写入 | 否 |
| 图片查看 | 本地文件系统 | 否 |
| Web界面 | 本地 uvicorn 服务器 | 否 |
| 定时同步 | 本地 APScheduler | 否 |

**无需钉钉开放平台Token、无需注册应用、无需付费、无需联网。**

### 4.2 钉钉桌面客户端的作用

- 本工具读取的是钉钉桌面客户端的**本地缓存数据库**
- 钉钉客户端必须在该电脑上**登录并使用过**，才会产生本地数据
- 运行本工具时**不需要**钉钉客户端在线（但数据库可能不是最新）
- 钉钉客户端在线时数据库通常使用 WAL 模式，本工具会复制 `dingtalk.db`、`dingtalk.db-wal`、`dingtalk.db-shm` 后再解密，尽量带上最新消息

### 4.3 数据完整性说明

- 本地数据库只包含该用户在该电脑上**看到过的**消息历史
- 如果某些群聊从未在该电脑上打开过，其消息可能不在本地数据库中
- 图片等附件需要钉钉客户端曾经加载过才会缓存到本地
- 未缓存的图片会显示"[图片未缓存到本地]"占位符
- 数据库更新频率取决于钉钉客户端的同步策略；若某个会话刚收到消息但本地还没落库，先在钉钉桌面端打开该会话，再执行一次手动同步

## 六、导出功能详解

### 6.1 导出操作步骤

1. 点击右上角 **"导出"** 按钮，打开导出弹窗
2. 在 **"勾选导出"** Tab 中勾选需要的会话（支持搜索、全选）
3. 在时间范围下拉框中选择：**全部 / 近3个月 / 近6个月 / 近1年 / 近2年**
4. 点击 **"导出选中会话"** 按钮
5. 等待导出完成后，自动切换到 **"已导出文件"** Tab
6. 点击 **"下载 ZIP"** 下载打包文件

### 6.2 导出目录结构

每次导出生成一个自包含目录：

```
export_20260417_191234/
├── export.json                # 消息数据（含附件相对路径引用）
├── images/                    # 所有图片文件
│   ├── 12345_67890.jpg
│   ├── 12345_67891_0.webp     # 引用消息中的图片
│   └── ...
└── files/                     # 所有文档附件
    ├── document_v1.0.docx
    ├── datasheet.pdf
    └── ...
```

### 6.3 导出 JSON 格式

`export.json` 中每条消息包含 `content` 字段，整合了文本和附件引用：

```json
{
  "message_id": 25760000918465,
  "sender_name": "UserA",
  "created_at_str": "2026-04-16 15:57:31",
  "content_type": 3100,
  "text": "[图片]\nsome description text",
  "content": "[图片: images/12345_25760000918465_0.png]\nsome description text",
  "image_export_paths": ["images/12345_25760000918465_0.png"]
}
```

**各消息类型的 `content` 字段格式：**

| 消息类型 | content 示例 |
|---------|-------------|
| 文本 | `那就从最低点量` |
| 图片(已缓存) | `[图片: images/12345_xxx.jpg]` |
| 图片(未缓存) | `[图片未下载到本地]` |
| 文件(已下载) | `[附件: document_v1.0.docx (737KB) \| files/document_v1.0.docx]` |
| 文件(未下载) | `[附件未下载: xxx.docx]` |
| 引用消息(含图片) | `[图片: images/xxx_0.webp]` `[图片: images/xxx_1.webp]` `引用的文本内容` |

### 6.4 供 AI Agent 使用

导出的 JSON 可供 AI 工具使用：
- `content` 字段将文本和附件路径整合为一条可读字符串
- 所有附件路径为相对路径，AI 可按路径读取实际文件
- 引用消息中的图片也能正确关联和导出
- ZIP 打包后可直接提供完整的知识库

## 七、常见问题

### Q：启动报错 "dingwave 不是内部或外部命令"
确认 `tools/dingwave.exe` 文件存在，或从 https://github.com/p1g3/dingwave/releases 下载。

### Q：同步失败 "Failed to copy encrypted database"
钉钉客户端正在锁定数据库文件。本工具会重试3次（间隔30秒）。如果持续失败，可以临时退出钉钉客户端后重试。

### Q：会话列表显示不完整
向下滚动会话列表即可自动加载更多（每次加载50个）。

### Q：会话列表太多，找不到目标会话
使用左侧搜索栏按名称搜索，或使用"全部/群聊/单聊"Tab按类型筛选。

### Q：某些图片显示"[图片未缓存到本地]"
该图片未被钉钉客户端缓存到本地（可能从未打开过该消息），本工具无法获取。

### Q：富文本/引用消息中的图片能显示吗？
能。工具会自动将富文本消息旁边的独立图片合并显示在文字中。

### Q：数据库只有V2格式吗？V3怎么办？
当前目录名中 `_v2` 表示V2加密。如果目标电脑目录名为 `_v3`，需要V3解密方式（需要 user_config 文件提取密钥），dingwave 目前不支持V3，需要额外开发。

### Q：导出的数据太大怎么办
可以使用时间范围筛选（选择"近3个月"等），只导出指定时间段的消息。也可以只勾选需要的会话导出。

### Q：导出文件如何下载
导出完成后，在"已导出文件"Tab中点击"下载 ZIP"即可下载包含所有附件的压缩包。

### Q：引用消息中的图片能导出吗
能。工具会自动查找引用消息在数据库中对应的所有图片文件，一并导出到 `images/` 目录，并在 `content` 字段中标注路径。

### Q：如何设为开机自启
创建快捷方式，目标设为：
```
python "/path/to/dingtalk-exporter/main.py"
```
放入 `shell:startup`（启动文件夹）中即可。

### Q：端口被占用怎么办
修改 `config.py` 中的 `WEB_PORT` 为其他端口号（如 `8091`）。
